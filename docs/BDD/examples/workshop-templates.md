### 📖 情報準備
- [ ] 対象機能の既存仕様書

### Story テンプレート

```markdown
## Story 作成テンプレート

### 基本フォーマット
📗 [ユーザー種別]として、[価値・目的]を得るために、[機能・行動]をしたい

### 子ども向けプロダクト特化版
📗 [年齢]歳の子どもとして、[学習目標・成長]を達成するために、[学習活動・体験]をしたい

### 記入例
📗 4歳の子どもとして、コンピューターに親しみを持つために、
   楽しいキャラクターと一緒にマウス操作を学びたい

### 検証ポイント
- [ ] 年齢が具体的に設定されている
- [ ] 教育的価値が明確になっている
- [ ] 子どもの視点で書かれている
- [ ] 測定可能な成果が含まれている
```

### Rule テンプレート

```markdown
## Rule 作成テンプレート

### カテゴリ別チェックリスト

#### 🛡 安全性・適切性
- [ ] 暴力的・恐怖的表現の排除
- [ ] ネガティブ表現の代替案
- [ ] 年齢適応性の確保
- [ ] 文化的配慮の組み込み

#### 🎯 ユーザビリティ
- [ ] タッチターゲットサイズ（44px以上）
- [ ] 操作ステップ数の制限（3ステップ以内）
- [ ] 視覚的フィードバックの提供
- [ ] 直感的操作の設計

#### ♿ アクセシビリティ・発達特性への包括的配慮

##### 発達障害への具体的配慮
- [ ] **ADHD**: 集中時間を短く区切る（5-10分）、注意散漫防止設計
- [ ] **ASD**: 予測可能なパターンとルーチンの提供、変化への事前準備
- [ ] **LD**: 多感覚的アプローチ（視覚+聴覚+触覚）、理解スタイル別対応
- [ ] **DCD**: 微細運動困難への配慮、大きなタッチターゲット

##### 感覚過敏への対応
- [ ] **視覚過敏**: 明度・コントラスト調整、点滅・急速な変化の回避
- [ ] **聴覚過敏**: 音量調整、突然の音の排除、無音モード提供
- [ ] **触覚過敏**: デバイス材質・温度への配慮、触感設定の選択肢

##### 運動機能の個人差への配慮
- [ ] **微細運動**: 大きなボタン・余裕のある配置（44px以上）
- [ ] **粗大運動**: 安定した姿勢での操作を可能にする設計
- [ ] **協調運動**: 同時操作の最小化、段階的な動作習得支援
- [ ] **疲労への配慮**: 短時間での完結、休憩タイミングの提案

##### 認知処理の多様性への対応
- [ ] **処理速度**: 個人ペースでの進行、時間制限なし
- [ ] **作業記憶**: 情報の分割提示、視覚的リマインダー
- [ ] **実行機能**: ステップの明確化、進行状況の可視化

##### 支援技術との互換性
- [ ] スクリーンリーダー対応
- [ ] スイッチコントロール対応
- [ ] 音声入力対応
- [ ] 視線入力システム対応

#### 👨‍👩‍👧‍👦 保護者連携
- [ ] 進捗の可視化
- [ ] 安全な学習環境の保証
- [ ] 適切な学習時間の管理
- [ ] 親子協働の機会提供

### Rule 記入フォーマット
📘 [条件・状況]の場合、[結果・動作]となる

### 記入例
📘 子どもが正しい操作をした場合、励ましのメッセージと楽しい効果音で成功を祝う
📘 操作に迷った場合、ヒントを表示するが決して「エラー」という言葉は使わない
```

### Example テンプレート

```markdown
## Example 作成テンプレート

### フォーマット
📒 [具体的状況] → [期待される結果]

### カテゴリ別Example

#### ✅ 成功パターン
📒 マウスを正確にクリック → 「すごいね！」メッセージ + キラキラエフェクト
📒 適切な速度でマウス移動 → キャラクターが同じ方向に滑らかに移動

#### 🤔 困った時パターン
📒 10秒間無操作 → 「ゆっくりやってみよう」ヒント表示
📒 マウスが動かない → 正しい持ち方アニメーション表示

#### 👨‍👩‍👧‍👦 保護者連携パターン
📒 学習セッション完了 → 保護者向け進捗レポート表示
📒 難しい操作で困る → 「おうちのひとにきいてみよう」表示

#### 🔧 技術的制約パターン
📒 ネットワーク接続不良 → オフラインモード自動切り替え
📒 デバイス性能不足 → 軽量版コンテンツに自動調整
```

### Question テンプレート

```markdown
## Question 整理テンプレート

### カテゴリ別分類

#### 📚 教育的効果
❓ どの程度の時間で基本操作を習得できるか？
❓ 年齢別の学習スピードの違いは？
❓ 学習効果の測定方法は？

#### 🔧 技術的実装
❓ マウス操作の精度をどこまで求めるか？
❓ 複数デバイス対応の優先度は？
❓ パフォーマンス要件の具体的数値は？

#### 👨‍👩‍👧‍👦 保護者ニーズ
❓ 保護者はどの程度の詳細情報を求めるか？
❓ 学習時間の制限機能は必要か？
❓ 進捗共有の頻度はどの程度が適切か？

#### 💼 ビジネス要件
❓ 競合との差別化ポイントは何か？
❓ 収益化の方法は？
❓ スケールアップ時の課題は？

### Question の優先度付け
🔴 高優先度: 実装開始前に解決必須
🟡 中優先度: 実装中に解決可能
🟢 低優先度: 将来的な検討事項
```

### 子ども向け表現集

```markdown
## 推奨表現 vs 避けるべき表現

### ✅ 推奨される表現
- 「すごいね！」「よくできたね！」
- 「もういちど やってみよう」
- 「ゆっくり やってみよう」
- 「いっしょに がんばろう」
- 「たのしく まなぼう」

### ❌ 避けるべき表現
- 「エラーが発生しました」
- 「操作に失敗しました」
- 「間違っています」
- 「システム障害」
- 「不正な入力」

### 🔄 言い換え例
| 避けるべき表現     | 推奨表現                     |
|-------------------|------------------------------|
| エラーが発生      | もういちど やってみよう      |
| 失敗しました      | もうすこし がんばってみよう  |
| 間違っています    | ちがう みちを えらんでみよう |
| 無効な操作        | べつの ほうほうを ためそう   |
| システム障害      | いま つながりにくいよ       |
```

## 3. ワークショップ進行テンプレート

### タイムテーブル（90分版）

```markdown
## Example Mapping ワークショップ（90分）

### 📋 開始前（5分）
- [ ] 参加者紹介
- [ ] ワークショップの目的説明
- [ ] ルール確認（一人一回発言、批判禁止等）

### 📗 Story 明確化（15分）
- [ ] 対象機能の背景説明（5分）
- [ ] Storyカード作成（10分）
- [ ] 全員でStoryの合意形成

### 📘 Rule 洗い出し（30分）
- [ ] 個人でRule案作成（10分）
- [ ] 全員でRule共有（15分）
- [ ] 重複排除・優先度付け（5分）

### 📒 Example 具体化（30分）
- [ ] 各Ruleに対するExample作成（20分）
- [ ] Example の妥当性確認（10分）

### ❓ Question 整理（10分）
- [ ] 不明点・課題の洗い出し
- [ ] 優先度付けと担当者決定

### 📋 まとめ・次のアクション（10分）
- [ ] 成果物の確認
- [ ] 次回までのアクション決定
- [ ] 成果物の写真撮影・記録
```

### ファシリテーション Tips

```markdown
## ファシリテーション注意点

### 👥 参加者管理
- 発言機会の均等化
- ドメインエキスパートの意見を重視
- 技術的実装よりもユーザー価値に焦点

### ⏰ 時間管理
- 議論が脱線したら軌道修正
- 詳細議論は別途実施を約束
- 必ず時間通りに終了

### 🎯 品質確保
- 曖昧な表現は具体化を促す
- 実際の子どもの行動を想像させる
- 保護者の視点も忘れずに確認

### 📝 記録方法
- 写真撮影（全体＋詳細）
- 主要な議論ポイントをメモ
- 決定事項と保留事項を明確に区別
```
